# Ex10 - The Art of the Prompt

First impressions
Take some notes during the presentations, paying attention to
what the prompting techniques evoke in you. Are you surprised,
skeptical, curious to try them? What catches your attention?

Emotion-Prompting -> easy & fast -> I kinda already do it
Plan-and-Solve -> make the steps for the LLM
Role Prompting -> LLMs can "act as"
-> ex: marketing & customers' targeted (Gwyneth)
S2A -> biases are inherent to everything + even AI is biased
+ might not be useful for everything like no need to unbias an art
though might be helpful to emphasize the opinions/irrelevances -
Thread-of-thought : helps to break a complex intentions into easier 1 -
- just one phrase to help enhance the whole results -
Reverse chain of thought : need to correct the prompt
Rephrase & respond : works well w humans & for difficult tasks for LLMs.
- easy bc just need to add "rephrase & respond" at the end of the prompt.
- helps to see if the machine understood the q°
Step back prompting : useful when questions with lots of details
-> go back to principles to answer properly (very much like human)
BUT need to provide the principles / context that the LLM needs to process
Chain of verification : ask a question to the LLM and then ask it to
check its own answer & if some were wrong, eliminate/correct
pb: if I already have a doubt on the LLM capacity -> might not use it.
+ might be long to implement
Self-Refine : looks like a dialog where you point specific issues
-> not really a technique + requires you to know there is a mistake
-> don't always correct the results. -> works w domain experts.
Self-Ask : improvement over Chain-of-Thought technique
-> need to add "do you think there are intermediary q° you need
to answer first" after the prompt? -> pretty useless in gl.
-> time consuming
